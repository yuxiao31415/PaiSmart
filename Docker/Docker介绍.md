## Docker 的核心工作原理  
**本质**:一个被隔离 + 限制资源的普通进程    
Docker 会隔离：  
* 进程（PID）  
* 网络（Network）  
* 文件系统（Mount）  
* 用户（User）  
* 主机名（UTS）  
## Docker 网络模型  
```
宿主机 (Host)
   |
docker0 网桥
   |
容器1   容器2   容器3
```
每个容器：

* 有自己的 IP（比如 172.17.x.x）
* 彼此可以通信
## Docker 三种核心网络模式

| 模式         | 关键词      | 是否需要端口映射 | 使用场景     |
| ---------- | -------- | -------- | -------- |
| bridge（默认） | NAT + 网桥 |  需要     | 单机开发最常用  |
| host       | 共用宿主机网络  |  不需要    | 高性能场景    |
| overlay    | 跨主机通信    | 看情况      | 集群 / 微服务 |
